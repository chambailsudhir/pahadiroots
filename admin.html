Production security — service key lives only in Netlify, never in browser

1️⃣ Netlify Environment Variables
Netlify Dashboard → Your site → Site Configuration → Environment Variables → Add variable
Variable name
SUPABASE_URL
Value
https://ulyrhnpoiypuvaurlqqi.supabase.co
Variable name
SUPABASE_SERVICE_KEY
Value
Supabase Dashboard → Project Settings → API → service_role (secret) key
Variable name
ADMIN_PASSWORD
Value
Your chosen admin password — you type this at the login screen
✅ Why this is safe on Netlify:
· Service key = Netlify server env var → never in browser JS
· Admin page calls /.netlify/functions/admin-api
· Function checks your password before touching DB
· Even if someone finds admin.html — they cannot do anything without the password
· Password is never stored — only in your session memory
2️⃣ Correct RLS Policies
Supabase → Authentication → Policies
products · states · categories
✅ anon: SELECT only (store can read)
🔑 service_role: all (admin writes, bypasses RLS)
orders · order_items
✅ anon: INSERT (customers can checkout)
✗ anon: no SELECT / UPDATE / DELETE
subscribers
✅ anon: INSERT (newsletter signup)
✗ anon: no SELECT / DELETE
